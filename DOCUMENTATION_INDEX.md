## 🛠️ Tools & Resources

### Useful Links
- **Google Gemini API**: https://makersuite.google.com
- **Usage Monitor**: https://makersuite.google.com/app/usage
- **API Keys**: https://makersuite.google.com/app/apikey
- **GitHub Issues**: https://github.com/mockmate/issues
- **Vercel Dashboard**: https://vercel.com/dashboard

### Command Reference
```bash
# Start development
npm run dev              # Start both servers
npm run dev:frontend    # Frontend only
npm run dev:backend     # Backend only

# Build for production
npm run build           # Build all
npm run build:frontend # Frontend only
npm run build:backend  # Backend only

# Testing
npm test               # Run test suite
npm run lint          # Check code style

# Database
npx prisma migrate    # Create new migration
npx prisma studio    # GUI database editor

# Deployment
vercel deploy         # Deploy to Vercel
```

### Key Files in Project
```
frontend/
  ├── src/lib/gemini.js              # AI service
  ├── src/lib/geminiKeyManager.js    # Key rotation
  ├── src/hooks/useGeminiApiStatus.js # Status monitoring
  └── src/lib/api.js                 # API client

backend/
  ├── routes/                         # API endpoints
  ├── prisma/schema.prisma            # Database schema
  └── server.js                       # Express server
```

## 💡 Best Practices

### Development
- ✅ Always check console for errors (F12)
- ✅ Use environment variables for secrets
- ✅ Commit documentation updates with code
- ✅ Test locally before deploying
- ✅ Keep dependencies updated

### Deployment
- ✅ Follow the deployment checklist
- ✅ Test in staging first
- ✅ Monitor for 24 hours after deploy
- ✅ Keep backups of database
- ✅ Rotate API keys monthly

### Operations
- ✅ Monitor API usage daily
- ✅ Check error logs weekly
- ✅ Review performance metrics
- ✅ Update dependencies monthly
- ✅ Rotate credentials regularly

## 📞 Getting Help

### Where to Look First
1. **This documentation** (you're here!)
2. **Browser console** (F12 → Console tab)
3. **FAQ & Troubleshooting** guide
4. **GitHub Issues** (might be solved already)

### When Stuck
1. Collect error information
2. Check [FAQ_TROUBLESHOOTING.md](./FAQ_TROUBLESHOOTING.md)
3. Search GitHub issues
4. Post on GitHub with details

### Info to Include When Asking for Help
- Your OS and Node version
- Exact error message
- Steps to reproduce
- What you tried
- Screenshots if helpful
