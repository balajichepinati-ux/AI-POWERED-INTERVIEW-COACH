## 🔄 API Endpoints Used

### New Endpoint
- **POST `/api/analyze/resume`** - Analyzes resume text
  - Input: `{ resumeText }`
  - Output: `{ analysis: { skills, technologies, targetRole, ... } }`

### Existing Endpoints Now in use
- **POST `/api/questions/generate`** - Generates questions
  - Input: `{ resumeData, category, count }`
  - Output: `{ questions: [...] }`

- **POST `/api/analyze`** - Evaluates answers
  - Input: `{ question, answer, category, resumeContext }`
  - Output: `{ analysis: { overallScore, clarityScore, ..., strengths, improvements } }`

## 🧠 Smart Fallbacks

The system is designed to be resilient:

**If Backend Fails:**
1. Resume Analysis → Falls back to client-side Gemini
2. Question Generation → Falls back to local question bank
3. Answer Analysis → Falls back to basic scoring + Gemini

**If Gemini API Keys Exhausted:**
1. Resume → Uses fallback extraction
2. Questions → Uses generic question bank
3. Answers → Uses demo scoring system

**Result:** System works even if APIs fail, though with reduced quality.

## 🔧 Troubleshooting

### Resume Upload Fails
**Check:** Network tab → `/api/analyze/resume` response
**Fix:** Verify backend is running and API key is valid

### Questions Are Generic
**Check:** SessionStorage → `mockmate_resume` has skills/tools
**Fix:** Resume analysis may have failed - upload again

### Answer Analysis Returns Error
**Check:** Backend logs for `[Gemini]` or `[Error]` messages
**Fix:** API key may be exhausted - rotate it!

### All Requests Return 500
**Check:** `backend/.env` exists and backend restarted
**Fix:** Run `echo $GEMINI_API_KEY` to verify env var is set
